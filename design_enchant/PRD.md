# Product Requirements Document (PRD)
## ระบบพอร์ตโฟลิโอสไตล์โปรแกรมเมอร์ (Programmer-Style CLI Portfolio)

---

## 1. ภาพรวมของระบบ (System Overview)

### 1.1 ที่มาและความสำคัญ (Background)
พอร์ตโฟลิโอรูปแบบเดิมที่เป็นเพียงหน้าเว็บแสดงผลข้อมูลสถิต (Static Resume) มักไม่สร้างความน่าสนใจให้กับผู้จัดหาบุคลากรด้านเทคนิค (Technical Recruiters) หรือผู้จัดการด้านวิศวกรรม (Engineering Managers) การสร้างพอร์ตโฟลิโอที่ผสมผสานเอกลักษณ์ของอินเทอร์เฟซแบบบรรทัดคำสั่ง (Command-Line Interface - CLI) แบบย้อนยุค (Retro Terminal) ร่วมกับกราฟิกสมัยใหม่ที่สวยงาม (Glassmorphism & Micro-animations) จะช่วยดึงดูดสายตาและสะท้อนทักษะการวิศวกรรมซอฟต์แวร์ได้อย่างเป็นรูปธรรม

### 1.2 แนวทางการออกแบบหลัก (Core Concept)
ระบบจะถูกพัฒนาขึ้นในลักษณะ **Dual-Mode Viewport** ที่ผสานการทำงาน 2 โหมดสลับกันได้อย่างสมบูรณ์แบบ:
- **Terminal Mode (โหมดบรรทัดคำสั่ง):** มอบประสบการณ์นักพัฒนาขนานแท้ มีเชลล์อินเทอร์แอกทีฟที่พิมพ์คำสั่ง, มีระบบประวัติคำสั่ง (Command History), ระบบสะกดคำ (Tab Completion) และการเล่นเกมสไตล์เรโทร (Snake, Typing)
- **GUI Mode (โหมดกราฟิกเว็บการ์ด):** อินเทอร์เฟซสไตล์โมเดิร์นกลาสมอร์ฟิซึม (Glassmorphism Cards) สำหรับบุคคลทั่วไปที่ไม่คุ้นเคยกับคำสั่งเทอร์มินัล ให้สามารถดูทักษะ ประสบการณ์ และติดต่อได้สะดวก

## 2. คุณสมบัติและความสามารถหลัก (Core Functional Requirements)

### 2.1 โครงสร้างเทอร์มินัลอินเทอร์แอกทีฟ (Interactive Terminal Shell)
- **Command Parsing & Command Registry:** ระบบถอดรหัสคำสั่ง (Tokenization) เพื่อเรียกใช้ฟังก์ชันย่อยตามคำสั่งที่ถูกลงทะเบียน
- **Tab Completion:** เมื่อผู้ใช้กดปุ่ม `Tab` ระบบจะเติมคำสั่งที่ตรงกับตัวอักษรตั้งต้นให้อัตโนมัติ หรือแสดงทางเลือก
- **Command History:** สนับสนุนการปัดปุ่มลูกศรขึ้น/ลง (`Up`/`Down` Arrow Keys) เพื่อเวียนดูคำสั่งที่เคยพิมพ์ไปแล้วในเซสชัน
- **Mobile Support:** แสดงแผงปุ่มเสมือน (Virtual Terminal Pad) บนสมาร์ตโฟน (เช่น `Tab`, `Arrow Up/Down`, `Esc`) และการแตะเทอร์มินัลเพื่อเรียกใช้คีย์บอร์ดมือถือได้อย่างราบรื่น

### 2.2 โหมดการแสดงผลแบบคู่ (Dual-Mode Transition)
- สามารถใช้คำสั่ง `gui` หรือกดปุ่ม Toggle สลับรูปแบบการแสดงผลระหว่าง Terminal และ GUI
- การสลับหน้าจอจะเกิดขึ้นพร้อมเอฟเฟกต์การเปลี่ยนผ่านหน้าจออย่างพรีเมียม (เช่น Code Rain Transition หรือ Screen Shatter Effect)

### 2.3 รายการคำสั่งของระบบ (Command Registry Dictionary)
ระบบรองรับคำสั่งพื้นฐานต่อไปนี้:
- `help`: แสดงตารางสรุปคำสั่ง คำอธิบาย และวิธีใช้
- `about`: แสดงประวัติย่อพร้อมภาพประกอบสไตล์ ASCII Art
- `projects`: แสดงผลโครงการพัฒนาตัวอย่าง (กรองด้วยธง `--tags` ได้)
- `skills`: แสดงแผนภูมิแท่งสไตล์เทอร์มินัลแสดงระดับความเชี่ยวชาญเทคโนโลยี
- `neofetch`: แสดงภาพอาร์ต ASCII โลโก้นักพัฒนาร่วมกับระบบปฏิบัติการและสถิติเว็บ
- `contact`: ระบบการส่งข้อความติดต่อโดยพิมพ์ฟิลด์แบบโต้ตอบทีละขั้นตอนผ่าน Shell
- `theme`: ปรับแต่งโทนสีของเทอร์มินัล (เช่น `dracula`, `matrix`, `amber`, `nord`)
- `matrix`: รันหน้าจอฝนสีเขียวตกลงมาสไตล์ภาพยนตร์ The Matrix
- `play <game>`: เล่นเกมย้อนยุคภายในเทอร์มินัล (เช่น `play snake`, `play typing`)
- `clear`: ล้างข้อมูลและประวัติการพิมพ์ออกจากหน้าจอ

## 3. หลักการออกแบบความสวยงาม (Design & Aesthetics)

1. **Aesthetics WOW Factor:** เลือกใช้พาเลตต์สีที่ออกแบบมาอย่างลงตัว หลีกเลี่ยงสีแดง/เขียว/น้ำเงินดิบ ให้ใช้ค่าสีแบบนีออนและพาสเทลสะท้อนแสงบนพื้นหลังสีดำเขม่า (Obsidian background)
2. **CRT Screen Simulation:** มีชั้นเอฟเฟกต์ลายเส้นสแกนหน้าจอย้อนยุค (CRT Scan Lines) และการกระเพื่อมของแสง (Flicker Effect) ที่ผู้ใช้เลือกเปิด/ปิดได้
3. **Typography Excellence:** บังคับใช้ฟอนต์ตระกูล Monospace ชั้นนำ เช่น `JetBrains Mono` หรือ `Fira Code` ในการแสดงผลทั้งหมดเพื่อความตรงของตัวอักษรทุกบรรทัด
4. **Micro-animations:** ติดตั้งอนิเมชั่นเคอร์เซอร์กะพริบจังหวะสมูท, การไล่สีปุ่มกดเมื่อนำเมาส์ไปชี้ (Hover transitions) และอนิเมชั่นการโหลดระบบเมื่อเข้าสู่หน้าเว็บครั้งแรก (Boot Sequence)

## 4. ความต้องการที่ไม่ใช่เชิงฟังก์ชัน (Non-Functional Requirements)

- **ความเร็วและประสิทธิภาพ (Loading Performance):** โครงสร้างโค้ดต้องถูกพัฒนาในรูปแบบ Next.js / TypeScript ที่โหลดได้รวดเร็ว ดึงข้อมูลแบบ Static Generation (SSG) เวลาการโหลดครั้งแรก (FCP) ต้องต่ำกว่า **1.5 วินาที**
- **การเพิ่มประสิทธิภาพการค้นหา (SEO Optimization):** จัดทำ Meta tag, Title tag ที่ช่วยให้แมตช์คำค้นหาใน Search Engine และนำทางด้วยโครงสร้าง HTML5 Semantic Elements (`<main>`, `<header>`, `<section>`, `<article>`)
- **การเข้าถึงระบบที่เท่าเทียม (Accessibility - a11y):** โทนสีทุกธีมต้องมีอัตราส่วนความต่างสี (Contrast Ratio) ที่อ่านง่าย และรองรับโครงสร้าง ARIA attributes สำหรับเครื่องอ่านหน้าจอ (Screen Readers)
- **ความเข้ากันได้ของเบราว์เซอร์ (Browser Compatibility):** ระบบแอนิเมชันและเอฟเฟกต์ Glassmorphism จะต้องแสดงผลได้เสถียรบน Chrome, Safari, Firefox และ Edge ทั้งบนคอมพิวเตอร์และมือถือ
