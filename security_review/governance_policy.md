# นโยบายการกำกับดูแล AI และกรอบความรับผิดชอบ (AI Governance & Accountability Policy)
## ระบบ Two-Layer Security Review (SAST & LLM Orchestration)

---

## 1. การคุ้มครองทรัพย์สินทางปัญญาและความลับข้อมูล (Data Privacy & IP Protection)

ข้อมูลโค้ดเบสขององค์กรถือเป็นหนึ่งในสินทรัพย์ที่มีมูลค่าและความลับสูงสุดของบริษัท นโยบายการใช้งานปัญญาประดิษฐ์จึงต้องมีข้อกำหนดที่เข้มงวดดังนี้:

- **การประมวลผลข้อมูลในระบบปิด (On-Premise / Private Cloud Only):** ข้อมูลซอร์สโค้ด, รายงานผล SARIF และบริบทการพัฒนาทั้งหมด ห้ามถูกส่งออกไปยังภายนอกบริษัทหรือ API สาธารณะ (เช่น OpenAI API, Anthropic API หรือบริการคลาวด์ภายนอกที่ไม่มีการทำข้อตกลงพิเศษ)
- **การไม่มีการบันทึกเพื่อฝึกฝนภายนอก (No Data Re-training Rights):** บริการประมวลผลโมเดลที่โฮสต์เองผ่าน vLLM ในเครื่องของบริษัทจะต้องถูกตั้งค่าไม่ให้มีการส่งข้อมูล telemetry หรือบันทึกเพื่อการทำ pre-training นอกเครือข่าย VPC
- **การจำกัดอายุข้อมูลชั่วคราว (Data Ephemerality):** ข้อมูล Code Diff และไฟล์ SARIF ที่ถูกส่งไปประมวลผลที่ vLLM/MCP Server จะต้องถูกลบออกจากหน่วยความจำชั่วคราวและดิสก์ประมวลผลทันทีหลังจากขั้นตอนการวิเคราะห์ใน Pipeline เสร็จสิ้น โดยห้ามเขียนข้อมูลโค้ดดิบแบบถาวร (Raw Code Persistence) ลงในฐานข้อมูลของบอทเด็ดขาด

---

## 2. ตารางขอบเขตความรับผิดชอบและการตัดสินใจ (Decision Authority Matrix)

เพื่อกำหนดขอบเขตและป้องกันปัญหาการโบ้ยความผิดพลาดให้แก่ระบบปัญญาประดิษฐ์ ให้แบ่งสิทธิ์และความรับผิดชอบตามเกณฑ์ดังนี้:

| บทบาทหน้าที่ | สิทธิ์ของ AI Bot | สิทธิ์ของนักพัฒนา (Developer) | สิทธิ์ของหัวหน้าทีม (Tech Lead / SA) |
| :--- | :---: | :---: | :---: |
| **การระบุช่องโหว่ความมั่นคง** | 🔴 เสนอแนะวิเคราะห์เชิงลึก | 🟢 รายงานและทบทวนความเสี่ยง | 🟢 ยืนยันความรุนแรงและแนวทางการป้องกัน |
| **การตัดสินใจอนุมัติ (Approve)** | ❌ **ไม่มีสิทธิ์โดยเด็ดขาด** | ❌ ไม่มีสิทธิ์อนุมัติโค้ดตนเอง | 🟢 **มีสิทธิ์อนุมัติการ Merge เข้าระบบหลัก** |
| **การข้ามการสแกน (Dismiss FP)** | ❌ ไม่มีสิทธิ์ลบประเด็นอัตโนมัติ | 🔴 เสนอขอละเว้นพร้อมเหตุผล | 🟢 **อนุมัติและยินยอมปิดประเด็นแจ้งเตือน** |
| **ความรับผิดชอบเชิงระบบ (Liability)** | ❌ **ไม่ต้องรับผิดชอบ** | 🟢 **รับผิดชอบร่วมต่อโค้ดเบสที่พัฒนา** | 🟢 **รับผิดชอบสูงสุดต่อคุณภาพของโค้ดเบส** |

> [!IMPORTANT]
> **หลักการพื้นฐานความรับผิดชอบ:** ปัญญาประดิษฐ์ในระบบนี้ทำหน้าที่เป็นเพียง "ที่ปรึกษาเชิงวิศวกรรม" (Advisory Assistant) เท่านั้น ไม่มีฐานะทางกฎหมายหรืออำนาจอนุมัติเชิงระบบ ความรับผิดชอบต่อความเสี่ยงบนระบบใช้งานจริง (Production) ยังคงผูกอยู่กับบุคคลที่เป็นผู้รีวิวและผู้กด Merge เป็นหลัก

## 3. ความโปร่งใสและการควบคุมกระบวนการ (Transparency & Auditability)

- **การบันทึกกิจกรรมเชิงออดิต (Compliance Logging):** ทุกการกระทำที่มีการละเว้นการแจ้งเตือนความปลอดภัย (Dismiss Alert) โดยระบุว่าเป็น False Positive (FP) จะต้องมีการเก็บประวัติระบุตัวตน ชื่อนักพัฒนา เหตุผลประกอบ และระดับความเสี่ยงของประเด็นดังกล่าวอย่างสม่ำเสมอใน Audit log
- **การสุ่มตรวจทานผลลัพธ์ (Periodic Quality Control):** ทีมผู้ประเมินคุณภาพซอฟต์แวร์หรือทีม Security จะต้องสุ่มดึงบันทึกการรีวิวของ AI และบันทึกการตัดสินใจของมนุษย์อย่างน้อยไตรมาสละครั้ง เพื่อประเมินผล:
  - มีช่องโหว่ใดที่เป็น True Positive แต่ AI ไประบุเป็น False Positive หรือไม่ (False Negative Leakage)
  - มีการตอบหลอกระบบเพื่อข้ามการสแกนความปลอดภัยโดยไม่สมเหตุสมผลหรือไม่ (Policy Compliance Bypass)
- **การปรับเปลี่ยนจริยธรรมการใช้งานโมเดล:** หากประสิทธิภาพหรือความเที่ยงตรงของ AI ลดลงต่ำกว่าเกณฑ์ความถูกต้องที่กำหนด (Accuracy < 80%) ทีมผู้ดูแลระบบมีหน้าที่ในการระงับการทำงานด่าน AI ชั่วคราว และเปลี่ยนผ่านกลับสู่ระบบตรวจสอบโดยคนทั้งหมดจนกว่าจะปรับปรุงเสร็จสิ้น
